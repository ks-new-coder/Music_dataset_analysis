# Music Store Data Analysis using SQL

## Overview
This project focuses on analyzing a music store dataset using SQL in PostgreSQL through pgAdmin.  
The main goal of this project is to explore customer behavior, sales performance, music preferences, and artist popularity by writing SQL queries and extracting meaningful insights from the data.

## Tools Used
- PostgreSQL
- pgAdmin
- SQL

## Dataset
The dataset represents a digital music store and includes information related to:
- Customers
- Employees
- Invoices
- Invoice lines
- Tracks
- Albums
- Artists
- Genres

## Objectives
- Analyze sales and customer purchase behavior
- Identify top-performing countries, cities, and customers
- Explore music genre preferences
- Find the best-selling artists and popular tracks
- Practice SQL concepts using a real-world relational dataset

## SQL Concepts Used
- SELECT
- WHERE
- ORDER BY
- GROUP BY
- LIMIT
- JOINS
- Aggregate Functions
- Subqueries
- Common Table Expressions (CTEs)
- DISTINCT

## Business Questions Solved
1. Who is the senior-most employee based on job title?
2. Which countries have the highest number of invoices?
3. What are the top 3 invoice totals?
4. Which city has generated the highest invoice total?
5. Who is the best customer based on total spending?
6. Which customers listen to Rock music?
7. Which artists have written the most Rock songs?
8. How much has each customer spent on the best-selling artist?

## Key Insights
- Identified the senior-most employee from the employee table.
- Found the countries with the highest invoice counts, showing the strongest customer activity.
- Determined the top invoice totals to understand high-value purchases.
- Analyzed city-wise invoice totals to identify top revenue-generating locations.
- Identified the highest-spending customer.
- Found customers who prefer Rock music.
- Ranked the top Rock artists by number of songs.
- Calculated customer spending on the best-selling artist using advanced SQL techniques like CTEs.
